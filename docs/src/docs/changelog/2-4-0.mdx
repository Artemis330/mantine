## Other changes

- [Center](https://mantine.dev/core/center/) component now supports `inline` prop to center inline elements
- Initial highlighted item in [Menu](https://mantine.dev/core/menu/) component no longer set on menu open â€“ user will have to use up or down arrow at least one to make selection visible
- [Card](https://mantine.dev/core/card/), [Paper](https://mantine.dev/core/paper/) and [Avatar](https://mantine.dev/core/avatar/) components now support setting custom component as root element via `component` prop
- [Text](https://mantine.dev/core/text/#dimmed) component now supports special `dimmed` color
- [Select](https://mantine.dev/core/select/), [Autocomplete](https://mantine.dev/core/autocomplete/) and [MultiSelect](https://mantine.dev/core/multi-select/) dropdowns are closed when input is clicked
- Components that support changing root element with `component` prop ([Button](https://mantine.dev/core/button/), [ActionIcon](https://mantine.dev/core/action-icon/), etc.) now have better TypeScript support
- [Loader](https://mantine.dev/core/loader/) component now supports 2 new variants
- [Select](https://mantine.dev/core/select/) component now supports `onSearchChange` event which allows to change data based on search query
- Indeterminate and checked icons were replaced in [Checkbox](https://mantine.dev/core/checkbox/) to make component more recognizable
