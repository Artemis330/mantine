## Usage

Input component is used as base for all other inputs (<GatsbyLink to="/core/select">Select</GatsbyLink>, <GatsbyLink to="/core/select">TextInput</GatsbyLink>, <GatsbyLink to="/core/select">Textarea</GatsbyLink> and others).
The single purpose of Input is to provide shared styles and features to other inputs.
Use other components listed above to build forms (as they provide better accessibility)
and Input as base for your own custom inputs with Mantine theme.

## Variants

Input has 3 variants, all of which are available on all Mantine inputs.
Note that, unstyled input variant may significantly impact usability, use it wisely.

## Get element ref

You can get input ref by passing `elementRef` prop to Input component:

```tsx
import React, { useRef } from 'react';
import { Input } from '@mantine/core';

function InputDemo() {
  const ref = useRef(null);
  return <Input elementRef={ref} />;
}
```

## TypeScript

Usage of some props with custom component in TypeScript require type specification.
Input component is a generic and you can pass 2 types to enhance type support:

```tsx
// Custom element button
<Input<'button', HTMLButtonElement>
  component="button"
  onClick={(event) => {}} // -> event will now have correct types
  elementRef={(node) => {}} // node type is set to HTMLButtonElement
/>
```

Usually it is not necessary to pass types and you can avoid it by setting types directly on callbacks:

```tsx
<Button
  component="button"
  onClick={(event: React.MouseEvent<HTMLButtonElement, MouseEvent>) => {}}
  elementRef={(node: HTMLButtonElement) => {}}
/>
```
