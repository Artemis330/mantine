## Usage with Portal

Usually, it is a good idea to render modal outside of current component to avoid styles collisions.
To implement this use <GatsbyLink to="/core/portal">Portal</GatsbyLink> component.

```tsx
import React, { useState } from 'react';
import { Modal, Portal } from '@mantine/core';

function PortalDemo() {
  const [opened, setOpened] = useState(false);

  return (
    <div style={{ position: 'relative', zIndex: 1 }}>
      <Portal zIndex={5}>
        <Modal opened={opened} onClose={() => setOpened(false)}>
          Modal is rendered at the end of document.body
        </Modal>
      </Portal>

      <button onClick={() => setOpened(true)}>Open modal</button>
    </div>
  );
}
```

## Change size

You can change modal width by setting `size` prop to predefined size or any valid width, for example, 55% or 200px.
Size controls modal width, max-width is set to 100% with spacing on left and right, no matter what `size` is passed 100% will not be exceed

```tsx
<Modal size="sm" /> // -> predefined small size
<Modal size={378} /> // -> size in px
<Modal size="55%" /> // -> size in %
<Modal size="calc(100vw - 87px)" /> // -> size with calc
<Modal size="255%" /> // -> max-width is set to 100%, won't work
```

<ModalSizesDemo />

Modal has 6 predefined sizes: xs, sm, md (default), lg, xl and full.
You can get these values by importing `MODAL_SIZES`:

```tsx
import { MODAL_SIZES } from '@mantine/core';
```

## Modal with overflow

You can control modal vertical overflow behavior by setting `overflow` prop:

- `outside` (default) – overflow is handled by modal wrapper
- `inside` – overflow is handled by modal body

<ModalOverflowDemo />

## Change transitions duration

You can change transitions duration by setting `transitionDuration` property. If you set it to 0, then there will be no appear and disapper animations.
Animations also disabled when user <GatsbyLink to="/hooks/use-reduced-motion/">prefers to reduce motion</GatsbyLink>.

<ModalTransitionsDemo />

## Accessibility and usability

- When modal is opened focus is trapped inside and document.body overflow is set to hidden
- When user clicks on overlay or presses escape modal closes
- Use `closeButtonLabel` prop to set `title` and `aria-label` attributes of close button for better screen reader support
- `aria-modal` is set to true on modal wrapper
- Modal wrapper has `aria-labelledby` and `aria-describedby` pointing to modal title and body
