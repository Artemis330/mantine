## Component state

In most cases you would use uncontrolled Accordion, to change initial opened item use `initialItem` prop:

```tsx
// -> Third element will be opened initially
<Accordion initialItem={2} />;

// -> No element will be opened initially (default)
<Accordion initialItem={-1} />;
```

If you need to manage component state on your own provide `state` and `onChange` props:

```tsx
import { useState } from 'react';
import { Accordion } from '@mantine/core';

function Demo() {
  // Second item will be initially opened
  const [state, onChange] = useState({ 0: false, 1: true, 2: false });
  return <Accordion state={state} onChange={onChange} />;
}
```

## Accessibility

Accordion component is accessible by default, it follows [WAI ARIA guidelines](https://www.w3.org/TR/wai-aria-practices-1.1/examples/accordion/accordion.html):

- All controls are included in `Tab` sequence
- `Up` and `Down` arrows move focus to previous/next control
- Item body has `role="region"` attribute
- Item control has `aria-expanded`, `aria-controls`, `aria-labelledby` attributes
- When user prefers to reduce motion, `transitionDuration` is automatically set to 0

To make component accessible for screen reader include descriptive in `Accordion.Item` label:

```tsx
<Accordion.Item label={<div />}>Content</Accordion.Item> // -> not ok
<Accordion.Item label="First item">Content</Accordion.Item> // -> ok
<Accordion.Item label={<div>First item</div>}>Content</Accordion.Item> // -> ok
```

## TypeScript

@mantine/core package exports following types associated with Accordion:

- `AccordionProps` – Accordion component props
- `AccordionItemProps` – Accordion.Item component props
- `AccordionState` – Accordion state type
- `AccordionStylesNames` – a union of styles names that can be used with [Styles API](/theming/styles-api/) (`classNames` and `styles` props)
