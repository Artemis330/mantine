use-form provides bare minimum api to manage simple forms.
It includes handlers to set and validate values.
Hook does not depend on `@mantine/core` inputs and does not work with dom.

```tsx
import { useForm } from '@mantine/hooks';
```

## Basic usage

Initialize form with initial values and optional validation rules. If you use TypeScript these initial values will determine form values types.

```tsx
const form = useForm({
  // initial values are required
  initiaValues: {
    email: '',
    termsOfService: false,
  },

  // validation rules are optional, add them if validation is required
  validationRules: {
    email: (value) => /^\S+@\S+$/.test(value),
  },
});
```

`form` object contains these values:

- `values` – fields values with the same shape as `initialValues` passed to useForm
- `errors` – object of booleans, indicates if field has validation error, has the same shape as `validationRules` passed to useForm
- `setFieldValue` – handler to set field value
- `setValues` – handler to set all form values
- `reset` – handler to reset form values to initialValues and set all errors to false
- `validateField` – handler to validate single field
- `setFieldError` – handler to set single field error
- `validate` – handler to validate all fields with given `validationRules`
- `onSubmit` – wrapper function for form onSubmit event

## TypeScript integration

use-form will use values types from initialValues, usually its better to initialize form with these placeholder values.
If it is not possible you can provide values type with generic:

```tsx
const form = useForm<{ name?: string; termsOfService?: boolean }>({
  initialValues: {},
});

// later
form.setValues({ name: 'name', termsOfService: false });
```
